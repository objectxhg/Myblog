2、然后在项目目录下新建一个SwaggerConfig 里面有两个方法

3、编写Controller 和 entity

3.1 Swagger2常用注解
@Api注解用于类 标识这个类是swagger的资源 填写模块名称、并扫描类里的所有接口生成文档

@ApiOperation用于方法说明；
@ApiImplicitParams 一般配合@ApiImplicitParam 使用
@ApiImplicitParam 指定单个参数的说明
@ApiModelProperty 实体类字段说明

4、现在可以启动项目访问 http://localhost:8088/docs.html 对应自己的端口号

如果访问出现404的情况 新建class 实现 WebMvcConfigurer，实现addResourceHandlers方法，设置资源路径映射。
（本文使用的是 swagger-ui-layer 所以不需要如下配置 如果使用的是原生的Swagger2 UI jar 添加如下配置即可）
就可以通过 http://localhost:8080/swagger-ui.html 正常访问了。
@Configuration //避免和其他组件冲突(例如：swagger2) 所以优先使用 实现 WebMvcConfigurer
public class WebConfig implements WebMvcConfigurer { // 单独使用 继承 WebMvcConfigurationSupport没问题

@Value("${upload.localUrl}")
private String localUrl;

@Override
public void addResourceHandlers(ResourceHandlerRegistry registry) {
      //这是我自己的项目的其他资源映射         
      registry.addResourceHandler("/upload/picture/**").addResourceLocations("file:///" + localUrl + "picture");
      registry.addResourceHandler("/upload/video/**").addResourceLocations("file:///" + localUrl + "video");

      /**
       * 只有使用swagger2原生的UI 才需要如下配置
       */
      //registry.addResourceHandler("/swagger-ui.html").addResourceLocations("classpath:/META-INF/resources/");
      //registry.addResourceHandler("/docs.html").addResourceLocations("classpath:/META-INF/resources/");

}
